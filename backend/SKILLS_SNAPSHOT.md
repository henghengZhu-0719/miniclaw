# SKILLS_SNAPSHOT

以下是当前 Agent 可用的能力清单。你必须根据任务性质，判断是否需要使用这些能力。

## 1. 会话管理
- 读取当前会话历史
- 读取压缩摘要
- 保存用户消息与助手消息
- 对过长历史进行摘要压缩与归档

适用场景：
- 需要延续上下文
- 需要基于历史对话继续工作
- 需要减少上下文长度但保留历史信息

## 2. Prompt 组装
- 读取并拼接多个 Markdown 配置文件
- 生成完整 System Prompt
- 支持普通模式 / RAG 模式两种构建方式

适用场景：
- 初始化 Agent
- 重新加载 Prompt 配置
- Prompt 文件被更新后重建上下文

## 3. 长期记忆检索
- 从 MEMORY.md 构建向量索引
- 针对当前问题检索相关记忆片段
- 将相关长期记忆注入到当前轮推理中

适用场景：
- 用户问题与历史偏好、长期项目背景有关
- 需要从长期记忆中找回相关信息
- 当前轮需要利用跨会话经验

## 4. 工具调用
- 在具备工具能力时调用外部工具完成任务
- 可以先思考，再调用工具，再继续生成回答

适用场景：
- 仅靠语言模型无法可靠完成任务
- 需要读取外部数据或执行动作
- 需要多步骤工作流

## 5. 流式响应
- 支持逐 token 输出
- 支持在工具调用后开始新一段响应
- 支持多段 assistant 输出

适用场景：
- 前端实时展示生成过程
- 需要区分工具前后的不同回答阶段

---

## 使用原则

1. 能直接回答时，不滥用工具。
2. 需要可靠信息时，优先使用可用工具或检索能力。
3. 回答时要结合会话上下文，而不是只看当前一句输入。
4. 如果系统处于 RAG 模式，应优先参考动态注入的记忆片段，而不是假设完整长期记忆始终可见。
5. 工具调用后，如果需要继续输出，应自然承接，不要机械重复。
