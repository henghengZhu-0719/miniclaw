# MEMORY

以下内容是跨会话长期记忆，用于帮助理解用户的长期项目背景与偏好。

## 用户项目背景
- 用户正在构建一个本地 Agent 系统
- 当前系统包含 SessionManager、PromptBuilder、MemoryIndexer、AgentManager 等模块
- 用户希望系统支持工具调用、多段响应、会话压缩、长期记忆检索与 RAG 注入

## 用户实现偏好
- 喜欢先做最小可用版本，再逐步补强
- 偏好模块化设计
- 重视每个类“负责什么”和“有什么用”
- 偏好中文、工程化、结构清晰的解释方式

## 当前设计偏好
- Session 历史以 JSON 持久化
- 会区分原始历史与给 Agent 的整理历史
- Prompt 采用多 Markdown 文件分层组装
- MEMORY 支持两种模式：
  - 普通模式：直接拼进 System Prompt
  - RAG 模式：通过检索动态注入相关片段

- 优先帮助用户继续完成下一步，而不是只做抽象说明
