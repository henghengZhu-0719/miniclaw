## 1. 保存每轮对话

它会把每一条消息持久化到：

```text
sessions/active/<session_id>.json
```

比如用户说一句、助手回一句，都会通过 `save_message()` 追加进去。每条消息里会带上：

* `role`
* `content`
* `tool_calls`
* `created_at`

这样你的会话不会只存在内存里，服务重启后也还能恢复。

### 有什么用

这是整个聊天系统最基础的“记账本”：

* 前端刷新后还能拿到历史
* 后端下次调用模型还能接着聊
* 以后做调试、导出、归档都靠它

---

## 2. 读取“原始历史”

`load_session(session_id)` 读取的就是原始 JSON 数组，不做加工。

例如可能长这样：

```python
[
  {"role": "user", "content": "你好"},
  {"role": "assistant", "content": "你好，我在。"},
  {"role": "assistant", "content": "请问要我做什么？"}
]
```

### 有什么用

这个主要给：

* 前端展示聊天记录
* 后端调试
* 归档逻辑
* 历史导出

也就是说，它保留的是“真实存储形态”。

---

## 3. 读取“给 Agent 用的历史”

`load_session_for_agent(session_id)` 不是原样返回，而是会做两件加工：

### 第一件：合并连续 assistant 消息

因为你的系统设计里，助手可能会出现多段输出：

* 工具前先说一段
* 调工具
* 工具后再说一段

存储层允许这些分段存在，但很多 LLM 更适合看到整理后的版本，所以这里会把连续 assistant 合并成一条。

### 第二件：把压缩摘要插到最前面

如果这个会话以前做过历史压缩，`summary.txt` 里有摘要，那么它会在最前面插入一条“虚拟 assistant 消息”，把之前压缩掉的上下文补回来。

### 有什么用

这是为了让模型看到的上下文更干净、更适合推理：

* 避免连续多条 assistant 破坏对话节奏
* 避免历史太长
* 又不丢失旧对话的重要信息

可以理解为：

> `load_session()` 给“存储/展示”用，
> `load_session_for_agent()` 给“模型推理”用。

---

## 4. 压缩旧历史

`compress_history(session_id, summary, n)` 会把前 `n` 条旧消息从 active 会话里拿出来：

* 追加到 `sessions/archive/<session_id>.jsonl`
* 把它们的摘要写到 `<session_id>.summary.txt`
* active 里只保留剩下的消息

### 有什么用

这是为了控制上下文长度。

因为聊天越久，直接把所有历史都喂给模型会有几个问题：

* token 越来越贵
* 响应越来越慢
* prompt 太长可能影响效果

所以要把早期历史“压缩成摘要”，以后只把摘要 + 最近消息给模型。

# 这个类在整个系统里的位置

如果把你的系统拆开看：

* `AgentManager`：负责调模型、调工具、流式输出
* `PromptBuilder`：负责组装 system prompt
* `MemoryIndexer`：负责 MEMORY.md 的向量检索
* `SessionManager`：负责会话历史的存储、读取、压缩

所以它更像是：

> **聊天系统的“会话仓库 + 上下文整理器”**

它不负责“怎么回答”，而是负责“模型回答时手里拿到什么历史”。

---

# 它具体解决了什么问题

你这个类主要解决 5 个现实问题：

### 1. 消息不能只存在内存里

否则服务重启就没了。

### 2. 前端和模型需要的历史格式不一样

前端想看原始分段消息；模型更适合看整理后的对话。

### 3. 多段 assistant 响应会打乱 LLM 输入

所以要合并。

### 4. 长对话会越来越贵

所以要压缩旧历史。

### 5. 被压缩掉的历史不能彻底丢

所以要有 `summary.txt` 和 `archive/`。

这些都是你这个类在兜底。

---

# 一句话理解每个方法

你现在这几个方法可以这么记：

* `load_session()`
  **读原始聊天记录**

* `save_message()`
  **追加一条聊天记录**

* `get_compressed_context()`
  **读历史摘要**

* `load_session_for_agent()`
  **把聊天记录整理成适合喂给模型的版本**

* `compress_history()`
  **把旧记录归档，并压缩成摘要**
